# Graphify Knowledge-Graph Integration Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development or superpowers:executing-plans to implement task-by-task. Steps use checkbox (`- [ ]`) syntax.

**Goal:** Stand up a committed, low-cost codebase knowledge graph via [`Graphify-Labs/graphify`](https://github.com/Graphify-Labs/graphify) and wire it into the `CLAUDE.md` workflow so it stays fresh and gets consulted as the codebase grows.

**Architecture:** Graphify is an AI-assistant *skill* + Python CLI. Code/SQL/Markdown are parsed locally with tree-sitter (no API cost). The queryable artifact `graphify-out/graph.json` is committed and versioned; a git hook rebuilds it (AST-only) on commit; agents query it via the `graphify` CLI documented in `CLAUDE.md`.

**Tech Stack:** `graphify` CLI (PyPI package name **`graphifyy`**), Python 3.10+ via `uv` (or `pipx`), tree-sitter (bundled). No LLM backend configured (AST-only ⇒ no API key, no cost).

## Global Constraints

- **Decisions (locked):** AST-only extraction to start (no LLM/API cost); **commit `graphify-out/graph.json`**, gitignore `graph.html`/`GRAPH_REPORT.md`; consumption via **CLI** (`graphify query/path/explain`) documented in `CLAUDE.md` — no MCP server in v1.
- CLI install package is **`graphifyy`** (double `y`); the invoked binary is `graphify`.
- **AST-only is the default** when no API key is set — no `--no-llm` flag exists or is needed; do not configure `ANTHROPIC_API_KEY`/`GEMINI_API_KEY`/`OPENAI_API_KEY` for graphify.
- Every doc/context change follows the mandatory Context Maintenance protocol (root `CLAUDE.md`): register new files in `00-Context-Map.md`, add a `DECISIONS.md` row, ISO-date changed rows, and `scripts/check-context-map.sh` must pass.
- Branch: `docs/graphify-integration`; do not merge to `main` directly; commit per-task.
- **Known caveats to document, not solve:** `.astro` is not among graphify's 36 tree-sitter grammars → `.astro` files graph partially (TS/JS/SQL/Markdown are fully covered). Git hooks are **local** (not committed) → each clone runs `graphify hook install`. The exact `.gitattributes` merge-driver entry graphify writes is undocumented → verify after `hook install`, don't hand-author.
