# Agent Rules — `app/src/db/`

Scope: DB client integration inside the application. Global app rules and the validation procedure live in `app/CLAUDE.md`. (2026-07-11)

## Rules

- Keep DB access compatible with Cloudflare Workers + Neon serverless.
- Use `DATABASE_URL_UNPOOLED` (direct, non-pooler) for the runtime DB client.
- Use `DATABASE_URL` (pooled) for migrations/seeds/introspection tools.
- Keep the DB client a small factory (`getDb()`), no global TCP pool logic.
- `app/src/db/schema.ts` is generated by `drizzle-kit introspect` — never hand-edit it. SQL migrations live in `database/`.
- Connection-variable semantics are owned by `docs/architecture/05-Database/11-Neon-Integration.md` §Connection String Rules — never redefine them here or in code comments beyond a pointer.
- You MUST NEVER use deprecated `neonConfig.fetchConnectionCache = true`

## Tool Allowances & Restrictions (2026-07-23)

Small, focused DB client integration layer. Most work is reading generated schema and the connection factory; rare edits to the factory function.

### Allow

- **Read** — load schema files, connection factory, Neon docs
- **Edit/Write** — modify the DB client factory
- **Bash** — run `drizzle-kit introspect`
- **Grep** — verify connection and schema usage patterns

### Restrict

Scoped to this layer's own work; skill-driven workflows use restricted tools as designed — see root `CLAUDE.md`.

- GitHub MCP tools (mcp__github__*) — DB layer changes don't require PR/issue interaction
- WebFetch, WebSearch — no external lookups needed
- Agent spawning — focused, contained work on the client factory
- Glob — limited file scope, not needed
